About Me

Hai, perkenalkan, saya Candora Sagala dan pendidikan terakhir S1 Prodi Teknik Informatika. Sejak tahun 2016 saya bergabung dengan PT Dipo Internasional Pahala Otomotif (DIP0) yaitu dealer resmi kendaraan bermotor merek Mitsubishi dengan jaringan dealer di wilayah Sumatera, Jawa, DKI Jakarta dan Kalimantan yang melayani penjualan unit kendaraan beserta layanan purna jualnya, termasuk penjualan suku cadang original, pelayanan bengkel dan perbaikan body kendaraan, yang berlokasi di kota Medan sebagai staff IT.

Berikut rincian pekerjaan yang pernah saya lakukan selama bergabung menjadi staff IT di DIPO :

  1. Membuat Aplikasi Cetak Giro : Sebuah aplikasi desktop yang berfungsi untuk mencetak giro menggunakan printer Dot Matrix dan dibuat menggunakan bahasa pemrograman C#.
  2. Admin Inventaris IT : mengelola proses pencatatan khususnya inventaris IT mulai dari pengajuan, pengadaan, pemeliharaan, retur hingga barang rusak. Dan setelah mengetahuin kondisi lapangan saya membuat aplikasi inventaris yang bisa mencatat semua informasi terkait inventaris tersebut mulai dari pembelian hingga barang rusak sehingga memudahkan pencarian history dan evaluasi pengadaan barang selanjutnya.
  3. Database Administrator : Bertanggung jawab terhadap backup, restore, repair database untuk semua server dealer yang bekerja secara remote. Dan juga bertanggung jawab terhadap persiapan dan implementasi system untuk dealer baru.
  4. IBS Sales Support : helpdesk untuk semua user IBS Sales yaitu sistem informasi penjualan kendaraan yang digunakan oleh DIPO Group.
  5. IBS Sales Developer : Melakukan perbaikan dan pengembangan system sesuai kebutuhan lapangan. Adapun tools yang digunakan adalah Visual Studio 2015 dengan bahasa pemrograman C# dan DBMS MySql.
  6. DMS (Dealership Managenet System) yaitu sistem baru yang akan digunakan DIPO sesuai arahan dari Mitsubishi Corp.
      Implementasi : melakukan meeting bersama vendor terkait DMS, melakukan meeting bersama internal (manajemen maupun pengguna IBS), mengajukan CR (Change Request) berdasarkan issue dilapangan.
      Data Migration and Cleansing : untuk kebutuhan penerapan DMS, sehingga data yang digunakan di sistem yang sedang berjalan (IBS) tetap bisa digunakan di DMS tanpa melakukan input ulang oleh user.
      Change Request : mengajukan change request terhadap BSI (Vendor yang di tunjuk Mistubishi Corp).
      Melakukan UAT : Melakukan Testing terhadap CR yang diajukan.
      DMS Key User Training : Melakukan pelatihan kepada user DIPO group terhadap penggunaan DMS Vehicle.
      Rekonsiliasi data IBS VS DMS : membuat rekonsiliasi, mendemokan ke user dan memantau serta memfollow up user untuk memastikan semua transaski yang terjadi di IBS sudah diinput dengan lengkap dan benar di DMS sehingga memudahkan pengecekan laporan tutup buku oleh pihak akunting.
